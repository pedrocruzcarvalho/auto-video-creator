# Codex Project Handoff

## Project

This repo is for **Extreme Survival**, a YouTube Shorts automation tool for fictional survival / almost-impossible situation videos.

Channel direction:

- Channel name: `Extreme Survival`
- Handle: `@ExtremeSurvivalTV`
- Tagline: `One situation. One rule. One way out.`
- Format: vertical YouTube Shorts, usually 25-35 seconds for the current pipeline.
- Niche: tricky survival simulations, impossible accidents, countdown tests, one-rule survival situations.

Do not revive the old documentary workflow. Do not mention or rebuild any Fern/documentary/archival pipeline. The current repo is intentionally Seedance-first.

## Critical Spend Rule

Do **not** run paid video generation unless Pedro explicitly asks for a generation run in the current chat.

Paid actions include:

- calling Replicate Seedance;
- calling Claude/Anthropic unless the user asks for Claude idea generation or clicks the app button themselves;
- running the Streamlit generate button through automation;
- any script path that invokes `run_seedance_native_pipeline`;
- any external AI video model test.

Safe actions without confirmation:

- reading/editing files;
- running `python -m py_compile`;
- running import/cost-estimate smoke checks;
- updating docs/prompts/UI;
- using the local Idea Lab;
- starting Streamlit locally, as long as no generation is triggered.

## Current Technical Workflow

Main pipeline:

- File: `pipeline/seedance_native.py`
- Model: `bytedance/seedance-2.0` through Replicate.
- Default output: two 15-second 9:16 clips.
- Clip 1: native Seedance voice/audio.
- Bridge: extract the final frame from clip 1.
- Clip 2: starts from clip 1 final frame. Do not send audio/video references together with the first-frame image because Seedance rejects that input combination.
- Final: stitch both clips, optionally transcribe audio with OpenAI Whisper, burn ASS captions with FFmpeg.

Main app:

- File: `streamlit_app.py`
- Run with: `streamlit run streamlit_app.py`
- App name: `Seedance Survival Studio`
- Built-in presets include `Sinking car water tank`, `Conveyor belt pulls you toward rollers`, `Walk-in freezer locks behind you`, and `Escalator step collapses under you`.
- The app also has `Idea Lab - no video generation`, which creates smart-survival idea presets. Local mode is free. Claude mode uses a small Anthropic text call and should only run when the user explicitly requests it or clicks the app button.
- Use `Custom` only when manually testing a new script and visual prompt.

CLI:

```powershell
python main.py "Sinking car water tank" --run-id seedance_sinking_car_v1
```

Useful options:

```powershell
python main.py "Sinking car water tank" --resolution 480p
python main.py "Sinking car water tank" --no-captions
python main.py "Sinking car water tank" --fresh
```

Outputs:

```text
output/<run_id>/
```

Important files:

- `final.mp4` - captioned final Short.
- `final_seedance_native.mp4` - stitched native-audio version.
- `native_transcript.txt` - transcript from audio.
- `contact_sheet.jpg` - review thumbnails.
- `run_report.json` / `run_report.md` - run details.

## Current Cost Assumptions

Estimated Seedance costs for two 15-second clips:

- `480p`: about `$2.70`
- `720p`: about `$6.00`
- `1080p`: about `$15.00`

Default recommendation:

- Use `720p` for serious tests.
- Use `480p` only for cheap prompt/style tests.
- Avoid `1080p` until a format is clearly working.

## Creative Direction

Visual style:

- semi-realistic viral 3D simulation;
- dynamic phone-first camera;
- constant push-ins, snap zooms, object close-ups, and camera movement toward the exact narrated action;
- bright, readable scenes;
- consistent main character and location across clips;
- no generated readable text, signs, letters, captions, numbers, logos, or UI inside the generated video.

Avoid:

- children-video cartoon style;
- dark horror;
- fake true stories;
- slow lore;
- real disaster reenactments;
- text generated by the video model.

Caption policy:

- The video model should not generate captions.
- Captions should be added in post with FFmpeg from the transcript.

Audio policy:

- Current best result uses Seedance native voice/audio.
- Captions are added afterward.
- Do not switch back to separate TTS unless specifically requested.
